OpenSidekick Settings

Bring your own LLM. Keys are stored locally in this browser and sent only to the provider you choose.

1. Providers

Add one or more model providers, then pick the one to use.

Need help choosing a model that works well?

OpenSidekick drives your browser by calling tools and (when Vision is on) looking at screenshots, so pick a model that supports both:

  • Tool / function calling — required. Without it the agent can't click, type, or navigate at all.
  • Vision (image input) — recommended. It's on by default. Without it, anything that needs to see the page (reading an image, a chart, a screenshot) won't work.

Two things trip people up: (1) within a family, often only a specific variant sees images — e.g. GLM's -V variant or DeepSeek's VL line, not the plain text model. (2) A model can support vision but have no tool-calling endpoint — some Qwen-VL endpoints on OpenRouter are like this, so they can see the page but can't act on it. You need both.

Easy picks — tested, tools + vision both work: the open qwen/qwen3-vl-30b-a3b-instruct is a fast, capable default (use qwen/qwen3-vl-235b-a22b-instruct for more muscle). meta-llama/llama-4-maverick, mistralai/mistral-small-3.2-24b-instruct and google/gemma-3-27b-it also pass. Prefer a closed model that just works? openai/gpt-4o.

Exploring on your own? Don't match on vision alone — open the OpenRouter models list, filter by Tools and image support, then paste the id into a provider below and hit Test to confirm both live. That beats any name we hard-code, which goes stale.

2. Behavior

In “Plan first”, the agent proposes its steps and the sites it will use, and waits for your approval before acting. Sensitive sites (banking, payments, crypto) and purchase/delete actions always confirm, regardless of this setting.

Lower = more focused and deterministic. Ignored by some models.

Lets the agent take a screenshot so a vision-capable model (Claude, GPT-4o / 4o-mini, Gemini, Llava…) can see the page — useful for images, canvas apps, or visual layout. On by default; turn off if you use a text-only model (it can't accept images and screenshots will fail).

Lets the agent run JavaScript in the page as an escape hatch (still permission-gated per site in “ask” mode). Powerful — enable only if you want it.

Adds read_console / read_network tools for debugging pages, via Chrome’s debugger. Chrome shows a “debugging this browser” banner while a task runs; it clears when the task ends.
